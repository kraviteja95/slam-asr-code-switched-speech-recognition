# SLAM-ASR: Code-Switched Hindi-English Speech Recognition

> End-to-end Automatic Speech Recognition (ASR) for **Hindi–English code-switched technical speech** using the **SLAM-LLM** paradigm: a frozen Whisper acoustic encoder + a small trainable projector + a LoRA-adapted 4-bit LLM decoder.
>
> Course project for *IIIT-Delhi — Building the Future of Voice & Audio (July 2026)*.

📦 **Repository:** [https://github.com/kraviteja95/slam-asr-code-switched-speech-recognition](https://github.com/kraviteja95/slam-asr-code-switched-speech-recognition)

## 1. Project overview

The **MUCS 2021 Subtask-2** dataset is Spoken-Tutorial recordings of Indian computer-science lectures where speakers freely mix Hindi (Devanagari) and English (Roman) — often *within* the same sentence. Standard ASR pipelines fail on this data because of:

- **Vocabulary explosion** — 17–33 % Out-Of-Vocabulary words on the blind-test set.
- **Script mixing** — the transcript must interleave Devanagari (`ट्यूटोरियल`) and Roman (`tutorial`).
- **Technical jargon** — code, numbers, acronyms and English proper nouns embedded in Hindi grammar.

This project builds a full SLAM-ASR pipeline from scratch, covering **every topic taught in the 5-day speech course**, and delivers a working demo evaluated by **Word Error Rate (WER)** after aggressive text normalisation.

```
Waveform (16 kHz) ──► 80-d log-Mel ──► Whisper-base encoder (frozen, 74 M)
                                          │
                                          ▼
                          1-D CNN (stride 5) + 2-layer MLP projector (trainable, ~20 M)
                                          │
                                          ▼
                       Prompt: "Transcribe the speech: <|audio|>×150"
                                          │
                                          ▼
                   Qwen-2.5-1.5B decoder (4-bit NF4) + LoRA (trainable, ~4 M)
                                          │
                                          ▼
                             Autoregressive Devanagari + Roman tokens
```

## 3. Rebuilding the LoRA weights (one-time, ~1 second)

The trained LoRA adapter is **70 MB**, but GitHub's web UI caps single-file uploads at 25 MB. It was therefore split into four ~18 MB parts. Before you can load the model, concatenate the parts back into a single `.safetensors` file.

### macOS / Linux

```bash
cd slam_asr_outputs/checkpoints/slam_asr/final/lora
cat adapter_model.safetensors.part_* > adapter_model.safetensors
# optional cleanup once verified:
# rm adapter_model.safetensors.part_*
```

### Windows (PowerShell)

```powershell
cd slam_asr_outputs\checkpoints\slam_asr\final\lora
Get-Content adapter_model.safetensors.part_* -Encoding Byte -ReadCount 0 |
    Set-Content adapter_model.safetensors -Encoding Byte
```

### Verify

The reassembled file must be exactly **73 911 112 bytes** (~70 MB) and loadable via safetensors:

```python
from safetensors.torch import load_file
weights = load_file('slam_asr_outputs/checkpoints/slam_asr/final/lora/adapter_model.safetensors')
print(f'Loaded {len(weights)} tensors')   # a few dozen keys expected
```

The same instructions are also inlined at [`slam_asr_outputs/checkpoints/slam_asr/final/lora/REASSEMBLE.md`](slam_asr_outputs/checkpoints/slam_asr/final/lora/REASSEMBLE.md).

## 5. Running on Kaggle

See **[`KAGGLE_SETUP.md`](KAGGLE_SETUP.md)** for a click-by-click walkthrough.

Short version:

1. Upload [`notebooks/slam_asr_all_in_one_kaggle.ipynb`](notebooks/slam_asr_all_in_one_kaggle.ipynb) to Kaggle.
2. Attach the **`kraviteja95/mucs-2021-hindi-english-code-switched-speech`** dataset.
3. Session options → **GPU T4 x2**, **Internet: On**.
4. Click **Run All** → ~2 hours end-to-end.

The dataset is already split into `train/`, `test/`, and `blindtest/` — no splitting logic lives in this repo. Any subsampling seen inside the notebook is purely for Kaggle's session time budget and is behind a `QUICK_MODE` flag at the top of Section 1.

For a **demo-only** session (no re-training), use [`notebooks/slam_asr_inferencing_demo_transcribe.ipynb`](notebooks/slam_asr_inferencing_demo_transcribe.ipynb) instead — it loads the checkpoint from a Kaggle Dataset and shows inline audio + transcripts in ~5 minutes.

## 6. Results

**Trained on:** 3 h subset of MUCS train (1 881 utterances, ≤15 s each), 1 epoch, ~236 optimizer steps, batch 2 × grad-accum 4 = effective batch 8. LoRA rank 16 on all `q,k,v,o,up,down,gate` projections of Qwen-2.5-1.5B in 4-bit NF4.

**Evaluated on:** 300-utterance samples of MUCS test and blindtest splits, with `CodeSwitchTextNormalizer` applied to both hypothesis and reference before `jiwer` computes WER/CER.

| Model | Trainable params | Test WER | Test CER | Blind WER | Blind CER |
|---|---:|---:|---:|---:|---:|
| Bi-LSTM + CTC (char) | ~4 M | 0.998 | 0.983 | — | — |
| Whisper-base zero-shot | 0 | 1.552 | 1.293 | — | — |
| Whisper-base fine-tuned (5 h) | 74 M | 3.929 | 3.232 | — | — |
| **SLAM-ASR (this project)** | ~24 M | **2.061** | **1.856** | **1.631** | **1.511** |

Per-language WER for SLAM-ASR: **Hindi 0.94, English 1.00** on test; **Hindi 0.96, English 1.00** on blindtest.

Raw numbers live in [`slam_asr_outputs/results/summary.json`](slam_asr_outputs/results/summary.json).

**Honest interpretation.** All four models fail this task in *different characteristic ways* (see [`docs/report_outline.md`](docs/report_outline.md) §9 for the full analysis). SLAM-ASR exhibits **mode collapse** — the projector, after only ~236 optimizer steps from a random init, produces near-identical audio embeddings regardless of input, so the LLM decoder emits a fixed seed sentence that repeats. This is a well-documented under-training pathology; reproducing the SLAM-LLM paper's numbers requires roughly 200× more compute (400+ GPU-hours vs our 1-2 GPU-hours).

The **pipeline works end-to-end** and is publicly reproducible — see `notebooks/`. Scaling to the full 90 h train set with 5–10 epochs is the direct route to competitive WER.

## 7. Reloading the trained model in code

After you've reassembled the LoRA safetensors (§3 above), the model loads with the exact same code as during training. Inside a Kaggle notebook that has bootstrapped the `src/` package:

```python
from src.inference.decode import load_slam_asr_from_checkpoint

model = load_slam_asr_from_checkpoint(
    'slam_asr_outputs/checkpoints/slam_asr/final',
    device='cuda',
)
# Whisper-base and 4-bit Qwen-2.5-1.5B are auto-downloaded from HuggingFace
# (~3 GB total). Only your ~91 MB LoRA + projector live on disk.
```

The full runnable example is [`notebooks/slam_asr_inferencing_demo_transcribe.ipynb`](notebooks/slam_asr_inferencing_demo_transcribe.ipynb).
